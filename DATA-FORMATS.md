# Data Formats

## Overview

FamilyNet VPN stores runtime data in JSON, SQLite and log files.
All runtime files live in `/data/` inside the container (`wg-vpn-data` volume).
Never commit these to Git.

---

## Runtime Files (inside container: /data/)

### api_token / api_tokens.json

Location: `/data/api_token` (legacy single token), `/data/api_tokens.json` (multi-token, v2 format)

Purpose: Recovery tokens for developer/SSH access. On startup, legacy `api_token` is migrated into `api_tokens.json` as "Recovery" token.

Format (api_tokens.json):
```json
[
  {"id": "migrated-0", "label": "Default", "token": "...", "created_at": 1718000000},
  {"id": "recovery", "label": "Recovery", "token": "...", "created_at": 1718000000}
]
```

Security: Secret. Never displayed on screen.

---

### clients.json

Location: `/data/clients.json`

Main client configuration file.

```json
{
  "server": {
    "privateKey": "...",
    "publicKey": "...",
    "address": "10.8.0.1"
  },
  "clients": {
    "u-u-i-d": {
      "name": "Admin",
      "address": "10.8.0.2",
      "privateKey": "...",
      "publicKey": "...",
      "preSharedKey": "...",
      "enabled": true,
      "role": "admin",
      "protected": false,
      "xray_uuid": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
      "created_at": "...",
      "updated_at": "..."
    }
  }
}
```

Fields:
- `xray_uuid` — VLESS UUID for Xray variant (generated for every peer regardless of variant)
- `protected` — prevents deletion/modification from UI
- `role` — `admin` (bypasses auth middleware) or `user`

Security: Contains private keys. Never commit.


---

### speed_limits.json

Location: `/data/speed_limits.json`

Per-client speed limit settings.

```json
{
  "u-u-i-d": {
    "rate": "10mbit",
    "enabled": true,
    "updated_ts": 1718000000
  }
}
```

Rate format: `^\d+(kbit|mbit|gbit|kbps|mbps|gbps)$`

---

### activity.log

Location: `/data/activity.log`

JSON Lines. Each line is valid JSON.

```json
{"ts":1718000000,"event":"create","target":"peer_name","detail":"..."}
```

Known event types:
- `create`, `delete`, `enable`, `disable`
- `name`, `role`
- `speed-limit`, `speed-normal`
- `maintenance`, `restart-vpn`, `restart-admin`, `reboot-server`
- `backup`, `restore-backup`, `restore-backup-started`
- `parental-set`, `parental-delete`
- `avatar`
- `error` (global exception handler)

---

```json
    "week_total_bytes": 987654321,
```

### traffic_stats.sqlite

Location: `/data/traffic_stats.sqlite`

Hourly and daily traffic history.

Schema:
```sql
CREATE TABLE traffic_hourly (
    peer_id TEXT,
    hour INTEGER,
    rx_bytes INTEGER DEFAULT 0,
    tx_bytes INTEGER DEFAULT 0,
    PRIMARY KEY (peer_id, hour)
);
CREATE TABLE traffic_daily (
    peer_id TEXT,
    day TEXT,
    rx_bytes INTEGER DEFAULT 0,
    tx_bytes INTEGER DEFAULT 0,
    PRIMARY KEY (peer_id, day)
);
```

## Backup Files

Location: `/data/backups/`

Files:
- `latest.wgadmin` — most recent backup
- `previous.wgadmin` — previous backup
- `YYYYmmdd_HHMMSS.wgadmin` — timestamped archives

Format: tar.gz.

Includes:
- clients.json
- settings.json
- speed_limits.json
- traffic_stats.json
- traffic_stats.sqlite
- activity.log
- health_state.json
- plus static web files

## Restore Safety Files

Location: `/data/restore-safety/`

Emergency snapshots created before restore operations.

## Xray Config (xray.json)

Location: `/data/xray.json`

Generated by `sync_xray_config()` in app_xray.py on startup and on every peer change.
Three inbounds: REALITY (443), WS (8443 internal / 8444 external), XHTTP (8445).

```json
{
  "log": {"loglevel": "warning"},
  "inbounds": [
    {
      "port": 443,
      "protocol": "vless",
      "settings": {"clients": [{"id": "...", "flow": "xtls-rprx-vision"}], "decryption": "none"},
      "streamSettings": {
        "network": "tcp",
        "security": "reality",
        "realitySettings": {
          "dest": "www.microsoft.com:443",
          "serverNames": ["www.microsoft.com"],
          "privateKey": "...",
          "shortIds": ["d64cc26c"]
        }
      }
    },
    {
      "port": 8443,
      "protocol": "vless",
      "settings": {"clients": [{"id": "..."}], "decryption": "none"},
      "streamSettings": {"network": "ws", "security": "none", "wsSettings": {"path": "/vless"}}
    },
    {
      "port": 8445,
      "protocol": "vless",
      "settings": {"clients": [{"id": "..."}], "decryption": "none"},
      "streamSettings": {"network": "xhttp", "security": "none", "xhttpSettings": {"path": "/vless"}}
    }
  ],
  "outbounds": [
    {"protocol": "freedom", "tag": "direct"},
    {"protocol": "blackhole", "tag": "block"}
  ]
}
```

Supporting files:
- `/data/reality_public` — REALITY public key (extracted from xray x25519)
- `/data/reality_short_id` — REALITY shortId (4-byte hex)

## WireGuard Config

Location: `/data/wg0.conf`

Generated by entrypoint.sh from clients.json.
Do not edit manually — overwritten on every container start and peer sync.

## Git Ignore Policy

Must be excluded:
- `api_token`
- `*.json` runtime files
- `*.sqlite`
- `*.log`
- `*.wgadmin`
- `*.tar.gz`
- `venv/`
- `__pycache__/`

May be committed:
- `common.py`, `app.py`, `app_xray.py`
- `web/*`
- `*.md`
- `Dockerfile`, `Dockerfile.awg`, `Dockerfile.xray`
- `entrypoint.sh`, `entrypoint-awg.sh`, `entrypoint-xray.sh`
- `install.sh`
- `requirements.txt`
- `init_config.py`
- `docker-compose.wg.yml`, `docker-compose.awg.yml`, `docker-compose.xray.yml`
- `.gitignore`
